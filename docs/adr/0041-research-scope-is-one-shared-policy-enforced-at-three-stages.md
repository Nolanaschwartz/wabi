# Research scope is one shared policy enforced at three stages; within-paper merge clusters in one call

The `@wabi/research` agent's definition of what is in scope — a **self-administered, non-clinical everyday practice** a person can do unaided (ADR-0001/0003) — lives in **one module** (`scope-policy.ts`) and is enforced at **three points** of the per-paper pipeline:

1. a deterministic, no-LLM **pre-screen** before the gate's model call, dropping clear supplement/drug/clinical abstracts for free;
2. the **gate** LLM call, which imports the shared scope fragment and judges topic-relevance against it;
3. the **judge**, which returns a separate `scopeOk` verdict and drops faithful-but-out-of-scope candidates.

All three import the **same prompt fragment** (`SCOPE_FRAGMENT`) so the wording can never drift between stages. Separately, **within-paper merge collapses a paper's lens candidates in a single clustering LLM call** behind the shared lexical prefilter, replacing the previous O(n²) pairwise adjudication loop.

## Why

- **Scope drift reached the review queue.** Production Langfuse traces showed faithful, high-scoring drafts that are out of scope for a non-clinical companion — "Take Vitamin D if Deficient" (0.9), evening tryptophan (0.85), protein timing (0.95). The gate was topic-blind and scope-permissive, the extractor had no scope constraint, and the judge scored faithfulness/quality but never scope. Each stage independently let scope-violating material through.
- **Defense-in-depth, not one chokepoint.** A single scope check would have to sit at one stage and would be one model call away from a regression silently reopening the leak. Putting scope at three stages of different *kinds* — a deterministic pre-screen (no model, no cost, catches the obvious), a topic-aware gate (cheap triage), and a judge verdict (final, per-candidate) — means a miss at one layer is caught by another, and the cheapest layer shoulders the obvious rejections so the model layers see fewer of them. The cost of three checks is one extra boolean in two existing prompts plus a pure function; the steps were already isolated single-purpose functions (ADR-0037), so this added no new pipeline structure.
- **One fragment, so the policy is honestly singular.** Three enforcement points are only safe if they enforce the *same* policy. A shared `SCOPE_FRAGMENT` string imported by gate, extractor, and judge makes "the scope policy" a single editable definition; without it, three hand-copied scope paragraphs would diverge on the first edit and the defense-in-depth would become three subtly different policies.
- **`scopeOk` is a separate verdict, not folded into the quality score.** The judge returns `scopeOk` as its own boolean so a trace records *why* a candidate was dropped (out of scope vs. unfaithful vs. below the tier floor). Collapsing scope into the 0–1 score would make a dropped supplement indistinguishable from a low-quality technique in the trace, defeating attribution (the same per-step-attribution principle as ADR-0040).
- **The pre-screen is conservative and fails open.** It rejects only on a clear out-of-scope signal (a vitamin/supplement/dosed-nutrient/drug/clinical keyword); everything ambiguous passes through to the topic-aware gate. This preserves fail-open mining (ADR-0021): the deterministic layer never silently drops a borderline paper — it only removes the unmistakable ones before they cost a model call.
- **Within-paper merge was firing 10–117 LLM calls per paper.** The old merge compared each candidate against every kept candidate via the pairwise duplicate check — O(n²) triage-LLM calls, the dominant per-run cost in the traces. The clustering call hands the model the paper's related-candidate list once and gets back groups, so merge makes **at most one** model call per paper. The shared lexical prefilter (the dedup Jaccard floor + normalization) still resolves the clear cases with no model at all: pairs at or above the ceiling auto-merge, pairs below the floor stay distinct, and only the ambiguous band reaches the call.
- **Batch clustering over tuning the pairwise prefilter.** We could have left the O(n²) shape and only widened the lexical prefilter band so fewer pairs escalated. Rejected: the prefilter is lexical and cannot tell genuine paraphrases ("box breathing" vs. "square breathing drill") apart from coincidence, so tightening it trades false-merges for missed-merges without removing the quadratic call count. One clustering call removes the quadratic factor *and* lets a single model judgment see all related candidates together, which is strictly more context than isolated pairwise comparisons.

## Considered options

- **One scope gate only** (no pre-screen, no judge verdict). Simplest. Rejected: a single stage is one prompt regression away from reopening the leak the traces already exhibited, and it spends a model call on supplements a keyword match rejects for free.
- **Fold scope into the judge's quality score** instead of a separate `scopeOk`. Fewer fields. Rejected: destroys drop-reason attribution in the trace and conflates "out of scope" with "low quality."
- **Pre-screen as an aggressive blocklist** that rejects on any nutrition/food mention. Rejected: would drop in-scope sleep-hygiene advice ("avoid caffeine before bed") and violate fail-open mining; the pre-screen stays conservative and leaves ambiguity to the gate.
- **Keep pairwise merge, only widen the lexical prefilter band.** Cheapest diff. Rejected: leaves the O(n²) call count and cannot distinguish paraphrase from coincidence lexically (see Why).
- **Embedding-based within-paper dedup** (vectors instead of a clustering LLM call). Most semantically accurate. Rejected here: personal/strategy embeddings live on the bot, not the worker (ADR-0009 store split); standing up an embedding path in the worker for within-paper merge is more infrastructure than one clustering call warrants. The lexical prefilter + single clustering call is the lazy correct fit, with embeddings as the documented upgrade path.

## Consequences

- **`scope-policy.ts` is the single source of truth for scope.** Gate, extractor, and judge import `SCOPE_FRAGMENT`; the orchestrator runs `prescreen` before the gate call. A scope-policy change is one edit in one file. The pre-screen's keyword list is a deliberate `ponytail:` ceiling — the gate LLM behind it is the real arbiter; the upgrade path if keywords prove too blunt is to lean harder on the gate, not to grow the list indefinitely.
- **The gate is topic-aware.** Its signature now takes the run topic; the orchestrator threads it through. The gate keeps its binary contract, temperature 0 (deterministic run-to-run), and fail-open policy.
- **Merge cost is bounded at ~one LLM call per paper**, down from up to 117. The lexical prefilter (ADR-shared with cross-paper dedup) is the shared floor/ceiling; a clustering error or unparseable reply keeps the lexical clusters un-merged (fail-open, ADR-0021) and never drops a candidate.
- **Out-of-scope material is dropped at three independent points**, so a regression in any one stage degrades gracefully rather than reopening the leak.
- **No change to the human-review boundary (ADR-0012).** Everything here happens before `StrategyDraft` submission; scope enforcement narrows what reaches the reviewer, it does not replace the review.
- **The evals (ADR-0040) gain new ground truth.** The gate and judge datasets should grow scope-labeled cases so a future prompt edit that weakens scope is caught offline.
